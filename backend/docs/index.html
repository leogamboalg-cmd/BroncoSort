Quick start

The default local base URL is http://localhost:3000. For a deployed server, replace it with the deployment URL. 

npm install
npm start

curl http://localhost:3000/api/health

Requests with a JSON body must include Content-Type: application/json. JSON request bodies are limited to 10 MB.

Common behavior

Authentication and CORS

The current API does not require an API key, bearer token, cookie, or user account from callers. Browser access is restricted by CORS to configured Chrome extension IDs, localhost, and configured university or community-college domains and their subdomains.

CORS is not authentication. curl, Postman, and server-to-server clients normally send no browser Origin header and are accepted by the current configuration.

Rate limits

Professor, school search, summary, school request, and DOM collection route groups allow 60 requests per client IP per minute. The checkout and utility routes do not use this shared limiter. 

{
  "error": "Too many requests, please try again shortly."
}

This response has status 429. The school-request endpoint also has its own per-IP and per-school limit.

Unhandled errors

Unexpected middleware errors return status 500:

{ "error": "Something went wrong internally." }
POST

/api/professor/ratings

Looks up one school and multiple professor names through Rate My Professors. Names are cleaned, deduplicated, and matched exactly after normalization. Accents, punctuation, capitalization, and hyphens do not affect the normalized comparison. Results are cached in Redis for 24 hours.

Request body

FieldTypeRequiredDescription
schoolstringYesSchool name to search
professorsstring[]YesOne or more professor names
 
{
  "school": "California State Polytechnic University-Pomona",
  "professors": ["Jane Doe", "John Smith", "TBA"]
}

TBA and To Be Announced are removed from names. Blank results are discarded and duplicate names are looked up only once.

JavaScript example

const response = await fetch(
  "http://localhost:3000/api/professor/ratings",
  {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      school: "California State Polytechnic University-Pomona",
      professors: ["Jane Doe", "John Smith"],
    }),
  },
);

const data = await response.json();
if (!response.ok) throw new Error(data.error);
console.log(data.ratingsByName);

Successful response

{
  "schoolFound": "California State Polytechnic University-Pomona",
  "ratingsByName": {
    "Jane Doe": {
      "found": true,
      "profName": "Jane Doe",
      "rating": 4.2,
      "numRatings": 38,
      "ranking": 125,
      "id": "1234567",
      "difficulty": 2.8,
      "percentTakeAgain": 84,
      "department": "Computer Science"
    }
  }
}

ranking is BroncoSort ranking data stored in Redis and can be null. If no exact professor match exists, the request still returns 200, but that professor has found: false, zero rating counts, and nullable details. An isolated lookup failure also adds "error": "Lookup failed" without preventing other names from succeeding.

Errors

404No matching schoolSchool not found

GET

/api/schools/search?q={query}

Searches the U.S. Department of Education College Scorecard by school name and returns at most 10 results. A missing query or fewer than two non-space characters returns 200 with an empty array. 

const query = new URLSearchParams({ q: "polytechnic pomona" });
const response = await fetch(
  `http://localhost:3000/api/schools/search?${query}`,
);
const schools = await response.json();
[
  {
    "id": 110529,
    "name": "California State Polytechnic University-Pomona",
    "city": "Pomona",
    "state": "CA",
    "website": "www.cpp.edu"
  }
]

Use URLSearchParams to encode user input. Dependency failures return 500 with School search failed; the shared limiter can return 429.

POST

/api/summary/getProfessorSummary

Finds an exact professor, reads up to 30 written reviews with an 80,000-character ceiling, and asks Gemini for a compact structured summary. Summaries are cached for 14 days.

This output is AI-generated from public reviews. Present it as a summary of opinions, not as verified fact.

Request body

{
  "school": "California State Polytechnic University-Pomona",
  "professorName": "Jane Doe"
}

Successful response

{
  "response": {
    "verdict": "Take Office Hours",
    "sentimentScore": 78,
    "tldrSummary": "Clear lectures and moderate projects, but strict grading makes careful preparation and regular attendance important.",
    "topPros": [
      "Clear lecture explanations",
      "Helpful office hours",
      "Useful project feedback"
    ],
    "topCons": [
      "Strict grading details",
      "Frequent weekly quizzes",
      "Fast lecture pace"
    ],
    "examStyle": "Quizzes and projects"
  },
  "cached": false
}

cached is true when Redis supplied an existing summary. sentimentScore ranges from 0 to 100. Pros and cons are intended to contain three short strings each.

Errors

  • 400 — Missing school or professorName
  • 404 — School not found
  • 404 — Professor not found
  • 404 — No written reviews found
  • 429 — shared rate limit exceeded
  • 500 — Server error fetching data
POST

/api/collect/store

Stores a school support request in Redis for 14 days and returns a random request ID. Only school and pages are checked for presence, but a useful submission should include all fields shown below. 

{
  "school": {
    "id": 123456,
    "name": "Example University",
    "website": "www.example.edu"
  },
  "pageUrl": "https://registration.example.edu/search",
  "pages": [
    {
      "name": "Class Search",
      "url": "https://registration.example.edu/search"
    }
  ]
}

The server stores extra supplied JSON fields. Do not submit secrets, student records, login information, or unnecessary page content.

Successful response

{
  "success": true,
  "requestId": "0d10a980-4eb1-4636-9940-a45026e50413"
}

One IP can submit at most three requests for the same school.id in 10 minutes. A fourth returns 429 with Too many requests for this school. Try again later.

Other errors

  • 400 — Invalid payload.
  • 400 — This school is already supported.
  • 500 — Failed to store school request.
POST

/api/collectData/collectData

Stores a sanitized structural sample of a registration page in Redis for seven days. A later submission for the same lowercased hostname replaces the earlier value.

Remove student names, IDs, course choices, tokens, form values, and private data before sending HTML. This endpoint is for structure, selectors, and counts.

Fields

FieldRequiredDefault
hostnameYesNone
skeletons.tableYesNone
pathnameNoEmpty string
capturedAtNoCurrent server time
detectionNoEmpty object
selectorHintsNoEmpty object
countsNoEmpty object
skeletons.movableBlockNoEmpty string
skeletons.groupContainerPreviewNoEmpty string
siblingSignaturesNoEmpty array

Example request

{
  "hostname": "registration.example.edu",
  "pathname": "/class-search",
  "detection": { "strategy": "table-headers" },
  "selectorHints": {
    "table": "#class-results",
    "instructor": ".instructor-name"
  },
  "counts": { "rowCount": 20, "siblingCount": 3 },
  "skeletons": {
    "table": "<table id=\"class-results\">...</table>",
    "movableBlock": "<div class=\"section-card\">...</div>",
    "groupContainerPreview": "<div id=\"results\">...</div>"
  },
  "siblingSignatures": [
    { "tag": "div", "className": "section-card" }
  ]
}

Successful response

{
  "message": "Structure received successfully",
  "school": "registration.example.edu",
  "saved": {
    "hasTable": true,
    "hasMovableBlock": true,
    "hasGroupContainerPreview": true,
    "siblingCount": 3
  }
}

Missing hostname or skeletons.table returns 400. Storage failures return 500. This route also uses the shared rate limiter.

POST

/api/checkout/create-checkout-session

Creates a one-time Stripe Checkout Session for one unit of the price configured by STRIPE_PRICE_ID. The request body is not currently read, so an empty object is sufficient. 

const response = await fetch(
  "http://localhost:3000/api/checkout/create-checkout-session",
  {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({}),
  },
);

const result = await response.json();
if (!response.ok) throw new Error(result.error);
window.location.assign(result.url);

Successful response

{
  "url": "https://checkout.stripe.com/c/pay/cs_test_example"
}

The success redirect is {FRONTEND_URL}/index.html?payment=success&session_id={CHECKOUT_SESSION_ID}. Cancellation redirects to {FRONTEND_URL}/index.html?payment=cancel.

A success redirect alone is not secure proof of payment. Fulfillment should verify the Stripe session or a Stripe webhook when implemented.

A missing price ID returns 500 with Missing STRIPE_PRICE_ID. Other Stripe failures return Failed to create checkout session. This endpoint has no shared API rate limiter.

Utility endpoints

GET /

Returns plain text: Hello from BroncoSort Backend!

GET /api/health

{
  "status": "active",
  "serverTime": "2026-06-22T18:30:00.000Z"
}

This confirms that the Express process responds. It does not verify Redis or any third-party integration.

GET /ip

{
  "ip": "203.0.113.10",
  "ips": ["203.0.113.10"],
  "header": "203.0.113.10"
}

This diagnostic endpoint shows Express's selected client IP, parsed proxy chain, and raw X-Forwarded-For header. The app trusts one proxy hop, which affects IP-based rate limiting.

Reusable frontend error handling

fetch does not throw just because a server responds with 400, 404, 429, or 500. Check response.ok explicitly. 

async function requestJson(url, options = {}) {
  const response = await fetch(url, options);

  let body;
  try {
    body = await response.json();
  } catch {
    body = null;
  }

  if (!response.ok) {
    throw new Error(body?.error ?? `HTTP ${response.status}`);
  }

  return body;
}

When receiving 429, wait before retrying. Do not retry in a tight loop. The standard limiter sends headers that can help a client decide when another attempt is appropriate.

Current limitations

  • There is no version prefix such as /api/v1.
  • There is no generated OpenAPI schema.
  • The API does not authenticate callers.
  • The checkout route has no endpoint-specific rate limiter.
  • Collection payloads only receive shallow validation.
  • Health checks do not verify external dependencies.
  • Some old standalone test scripts reference obsolete routes or payloads.

The endpoint paths on this page match the current route mounts in server.js and the current files in routes/.
